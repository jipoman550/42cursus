## 📝 Introduction
이 프로젝트는 C++에서 가장 중요한 **메모리 관리**의 기초를 다집니다. 단순히 객체를 만드는 것을 넘어, 메모리의 어느 영역(Stack vs Heap)에 할당할지 결정하고, 포인터의 발전된 형태인 **참조자(Reference)**와 **멤버 함수 포인터**의 개념을 학습합니다.

## 🛠️ General Rules
과제를 수행하며 반드시 준수해야 하는 엄격한 규칙들입니다. 이 규칙을 어기면 실력이 늘지 않을 뿐더러 점수도 처참해질 거야!

### 1. Compilation
* **Compiler**: `c++`
* **Flags**: `-Wall -Wextra -Werror`
* **Standard**: `-std=c++98` 플래그를 추가해도 완벽하게 컴파일되어야 합니다.

### 2. Naming Conventions
* **Directories**: `ex00`, `ex01`, ..., `exn`
* **Class Names**: **UpperCamelCase** 형식을 따릅니다.
* **Files**: 헤더 파일과 소스 파일명은 반드시 클래스 이름과 일치해야 합니다. (예: `Zombie.hpp`, `Zombie.cpp`)

### 3. Forbidden & Allowed
* **C 함수 사용 금지**: `*printf()`, `*alloc()`, `free()` 사용 시 **0점** 처리됩니다.
* **Keywords**: 명시적인 허용 없이는 `using namespace <ns_name>`과 `friend` 키워드를 사용할 수 없습니다. (위반 시 **-42점**)
* **STL 제한**: Module 08 이전까지 컨테이너(vector, list, map 등)와 알고리즘 헤더 사용이 금지됩니다. (위반 시 **-42점**)
* **메모리 관리**: `new`를 사용해 할당한 메모리는 반드시 `delete`로 해제하여 **Memory Leak**을 방지해야 합니다.

### 💡 Deep Dive

#### **Q1. 힙(Heap) 할당은 언제 사용하나요?**
* **답변**: 객체가 함수가 종료된 후에도 살아있어야 하거나, 컴파일 타임에 크기를 알 수 없는 큰 데이터를 다룰 때 사용합니다. `new` 키워드를 사용하며, 반드시 `delete`로 직접 수동 해제해주어야 합니다.

#### **Q2. 스택(Stack) 할당의 장점은 무엇인가요?**
* **답변**: 관리가 매우 쉽고 빠릅니다. 함수의 중괄호(`{ }`)를 벗어나는 순간 시스템이 자동으로 메모리를 회수하고 소멸자를 호출해주기 때문입니다. 메모리 누수 위험이 없어 가능하면 스택 할당을 우선적으로 고려하는 것이 좋습니다.

#### **Q3. 소멸자에서 이름을 출력하게 하는 이유는 무엇인가요?**
* **답변**: 프로그래머가 메모리 해제를 적절한 타이밍에 했는지 시각적으로 확인하기 위해서입니다. 힙에 할당된 좀비를 `delete` 하지 않으면 프로그램 종료 시까지 메시지가 출력되지 않으며, 이는 곧 메모리 누수를 의미합니다.

#### **Q4. `Zombie z(name);`와 `Zombie z = Zombie(name);`은 똑같은 건가요?**
* **답변**: 결과는 같아 보이지만 미묘한 차이가 있습니다. 첫 번째는 **직접 초기화(Direct Initialization)**로, 즉시 객체를 생성합니다. 두 번째는 **복사 초기화(Copy Initialization)**로, 임시 객체를 만든 뒤 이를 복사하는 과정을 거칠 수 있습니다. C++98 표준에서는 성능과 명확성을 위해 첫 번째 방식을 더 권장합니다.

#### **Q5. C언어의 `malloc`이 있는데 왜 C++에서는 `new`를 쓰나요?**
* **답변**: `malloc`은 단순히 메모리 덩어리만 할당할 뿐이지만, `new`는 연산자로서 **객체의 생성자를 자동으로 호출**해줍니다. 즉, 메모리 할당과 동시에 객체를 사용할 준비(초기화)까지 마쳐주기 때문에 C++의 객체 지향 철학에 필수적입니다. 반대로 `delete`는 메모리 해제 전 **소멸자를 호출**해 뒷정리를 도와줍니다.

#### **Q6. 함수가 끝났는데 `delete`를 안 하면 어떻게 되나요?**
* **답변**: 힙(Heap)에 할당된 메모리는 프로그래머가 직접 관리하기로 약속한 영역입니다. 따라서 `delete`를 호출하지 않으면 프로그램이 끝나도 해당 객체의 **소멸자가 실행되지 않으며**, 이는 메모리 누수(Memory Leak)로 이어집니다. "Destroyed" 메시지가 출력되지 않는다면 바로 이 상황입니다.

#### **Q7. 왜 함수 이름이 `randomChump`인가요?**
* **답변**: "Chump"는 엑스트라나 만만한 사람을 뜻하는 속어입니다. 이 함수는 좀비를 **스택(Stack)**에 생성하여 함수가 끝나면 바로 사라지게 만듭니다. 즉, 주인공처럼 계속 살아남는 좀비가 아니라 잠깐 나타났다 사라지는 '별 볼 일 없는 좀비'라는 의미를 할당 방식의 특성에 빗대어 위트 있게 표현한 것입니다.

#### **Q8. 생성자 본문(`{}`) 대신 `: name(name)`처럼 초기화 리스트를 쓰는 이유는 무엇인가요?**
* **답변**: C++에서 객체가 생성되는 과정은 '초기화'와 '대입'으로 나뉩니다.
* **효율성 (Initialization vs Assignment)**: 초기화 리스트(`: name(name)`)를 사용하면 객체가 메모리에 올라감과 동시에 값이 설정됩니다. 반면 `{ name = name; }`처럼 본문에서 대입하면, 일단 빈 값으로 객체가 생성된 후 다시 값을 덮어씌우는 과정을 거치므로 비효율적입니다.
* **필수적인 상황**: 다음과 같은 경우 초기화 리스트 사용이 선택이 아닌 **필수**입니다.
	1.  **`const` 멤버 변수**: 상수는 생성과 동시에 값이 정해져야 하며, 생성자 본문에서의 대입(수정)이 불가능합니다.
	2.  **참조자(`&`) 멤버 변수**: 참조자는 선언과 동시에 대상을 가리켜야 하므로 초기화 리스트에서 정의해야 합니다.
	3.  **기본 생성자가 없는 클래스 멤버**: 멤버 변수로 포함된 다른 클래스가 인자 없는 생성자를 지원하지 않을 때, 어떻게 초기화할지 리스트에 명시해야 합니다.
* **가독성**: 객체의 기반이 되는 데이터 설정을 로직 실행부(`{}`)와 분리하여 코드의 의도를 명확히 전달합니다.
```cpp
/* Zombie.cpp */

Zombie::Zombie(std::string name) : name(name) // 초기화 리스트: 탄생과 동시에 이름 부여
{
	// 생성자 본문: 초기화 이후 실행될 로직이 들어가는 곳
}
```

#### **Q9. `Zombie::Zombie(std::string name) : name(name)`에서 왜 같은 이름을 두 번 쓰나요?**

* **답변**: 이것은 C++에서 **멤버 변수**와 **매개변수**의 이름이 같을 때 발생하는 '이름 가리기(Name Shadowing)' 문제를 가장 깔끔하게 해결하는 방법입니다.
* **역할 분담**:
	* **앞의 `name` (콜론 뒤)**: 클래스 내부(`private`)에 선언된 **멤버 변수**를 가리킵니다. 컴파일러는 이 위치에 오는 이름을 무조건 멤버 변수로 인식하도록 약속되어 있습니다.
	* **뒤의 `(name)` (괄호 안)**: 생성자의 인자로 전달된 **매개변수**를 가리킵니다.
* **장점**:
	* `this->name = name;`과 같은 추가적인 대입 연산 없이, 객체가 생성되는 시점에 인자 값을 멤버 변수에 직접 주입합니다.
	* `newName`이나 `pName` 처럼 억지로 이름을 바꿀 필요가 없어, 변수의 의미(이름을 설정한다)를 더 명확하게 유지할 수 있습니다.

```cpp
/* Zombie.cpp */

// (std::string name) <- 외부에서 전달된 '매개변수'
Zombie::Zombie(std::string name)
	: name(name) // 앞의 name: 클래스의 멤버 변수 / 뒤의 (name): 받은 매개변수
{
	// 초기화 리스트를 통해 '내 주머니(멤버 변수)'에 '네가 준 것(매개변수)'을
	// 탄생과 동시에 넣었으므로, 이 중괄호 안은 비워두어도 됩니다.
}
```


### 🧠 메모리 구조 다시보기 (Memory Segments)

객체가 저장되는 위치를 이해하는 것은 C++ 설계의 시작입니다.

* **Stack**: 지역 변수, 매개변수. `{}`를 벗어나면 자동 소멸. (`randomChump`)
* **Heap**: `new`로 할당한 동적 메모리. `delete` 전까지 유지. (`newZombie`)
* **Data/BSS**: 전역 변수, 정적(static) 변수가 저장되는 공간. 프로그램 종료 시까지 유지.
