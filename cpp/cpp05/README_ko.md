# 📖 C++ - Module 05

**Repetition and Exceptions (반복과 예외 처리)**

## 📝 Introduction

이 프로젝트는 C++에서 런타임 오류를 우아하게 처리하기 위한 핵심 메커니즘인 예외 처리(Exception Handling)를 다룹니다. 객체가 스스로의 무결성을 보호하기 위해 유효하지 않은 상태가 되는 것을 거부하고, 발생한 문제를 호출자에게 알리는 `try-catch` 구조와 커스텀 예외 클래스 설계법을 학습합니다.

## 🛠️ General Rules

과제를 수행하며 반드시 준수해야 하는 규칙들입니다.

### 1. Compilation

* **Compiler**: `c++`
* **Flags**: `-Wall -Wextra -Werror`
* **Standard**: `-std=c++98` 플래그를 추가해도 완벽하게 컴파일되어야 합니다.

### 2. Forbidden & Allowed

* **C 함수 사용 금지**: `printf()`, `malloc()`, `free()` 사용 시 **0점** 처리됩니다.
* **STL 제한**: Module 08 이전까지 컨테이너(`vector`, `list`, `map` 등)와 알고리즘 사용이 금지됩니다.
* **사용 금지 키워드**: `using namespace`와 `friend` 사용 시 **-42점** 감점됩니다.
* **Orthodox Canonical Class Form (OCCF)**: 모든 클래스는 반드시 기본 생성자, 복사 생성자, 복사 대입 연산자, 소멸자를 포함해야 합니다.
* *단, 예외 클래스는 이 형식을 따르지 않아도 됩니다.*

## 🏛️ Exercise 00: Bureaucrat

가장 작은 톱니바퀴인 `Bureaucrat`(관료) 클래스를 통해 예외 처리의 기초를 다룹니다.

### Objectives

* 1(최고) ~ 150(최저) 등급을 가진 관료 클래스 설계.
* 범위를 벗어나는 등급 설정 시 예외를 던지는(throw) 로직 구현.
* `std::exception`을 상속받은 커스텀 예외 클래스 활용.

### 💡 Deep Dive

#### **Q1. `private`과 `public`은 정확히 어떤 역할을 하나요?**

* **데이터 보호(private)**: `_name`과 `_grade`를 숨겨 외부에서 `tom.grade = 0;`과 같이 규칙을 어기고 직접 수정하는 것을 방지합니다.
* **소통 창구(public)**: 사용자는 공개된 `incrementGrade()` 등을 통해서만 안전하게 등급을 조정할 수 있으며, 이 내부에서 유효성 검사가 이루어집니다.

#### **Q2. `std::string`은 클래스인가요?**

* **그렇습니다**: `std::string`은 메모리 관리를 스스로 수행하는 완성된 클래스입니다.
* **과제에서의 활용**: `const std::string _name;`으로 선언하여, 객체 생성 시 이름을 고정하고 이후 변경할 수 없도록 보장합니다.

#### **Q3. 커스텀 예외 클래스의 문법이 왜 이렇게 특이한가요?**

* **상속과 재정의**: `std::exception`을 상속받아 `virtual const char* what() const throw();`를 오버라이딩합니다.
* **`throw()`의 의미**: "이 함수는 내부에서 예외를 던지지 않는다"는 약속입니다. 이를 통해 예외 메시지를 반환하는 과정에서 또 다른 에러가 발생하는 것을 방지합니다.

#### **Q4. 생성자에서 `: _name(name), _grade(grade)` 형식을 쓰는 이유가 무엇인가요?**

* **멤버 초기화 리스트**: 객체의 몸체(`{}`)가 실행되기 전에 멤버를 초기화합니다.
* **상수(const) 초기화**: `_name`이 `const`이기 때문에, 몸체 안에서의 '대입'이 아닌 초기화 리스트를 통한 '초기화'가 문법적으로 필수입니다.

#### **Q5. `operator<<`를 왜 멤버 함수가 아닌 전역 함수로 만드나요?**

* **왼쪽 피연산자 문제**: `std::cout << bureaucrat`에서 연산자 왼쪽의 주인은 `std::cout`(ostream)입니다.
* **유연성**: 우리가 `std::ostream` 클래스를 수정할 수 없으므로, 외부에서 두 객체가 만났을 때의 행동을 정의해 주는 것입니다.

#### **Q6. 예외가 발생하면 프로그램은 어떻게 동작하나요?**

* **즉시 중단**: `throw`가 실행되는 순간, 그 아래의 로직(출력 등)은 무시되고 즉시 가장 가까운 `catch` 블록으로 점프합니다.
* **무결성 보장**: 생성자에서 예외가 터지면 객체 자체가 만들어지지 않습니다. 즉, "등급이 0인 위험한 관료"가 메모리에 존재할 가능성을 원천 차단합니다.

#### **Q7. `std::cout`과 `std::cerr`의 차이는 무엇인가요?**

* **용도 분리**: 일반적인 정보 출력에는 `std::cout`을, 에러나 예외 상황 전파에는 `std::cerr`을 사용합니다.
* **버퍼링**: `cerr`은 버퍼를 거치지 않고 즉시 출력되므로, 프로그램이 예기치 않게 종료되더라도 에러 로그를 확실히 남길 수 있습니다.
