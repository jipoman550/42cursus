# 📖 C++ - Module 02
**Ad-hoc polymorphism, operator overloading and Orthodox Canonical class form**

## 📝 Introduction
이 프로젝트는 C++의 핵심 개념인 **다형성(Polymorphism)**, **연산자 오버로딩(Operator Overloading)**, 그리고 객체의 복사와 대입을 안전하게 다루기 위한 **정통 표준 클래스 형태(Orthodox Canonical Class Form)**를 학습합니다. 컴퓨터 과학에서 실수를 표현하는 방식인 부동 소수점(Floating-point)의 한계를 이해하고, 이를 보완할 수 있는 **고정 소수점(Fixed-point)** 클래스를 직접 구현하며 내부 동작 원리를 깊이 있게 파헤칩니다.

## 🛠️ General Rules
과제를 수행하며 반드시 준수해야 하는 규칙들입니다. Module 02부터는 클래스 설계에 있어 아주 중요한 새로운 규칙이 추가됩니다.

### 1. Compilation
* **Compiler**: `c++`
* **Flags**: `-Wall -Wextra -Werror`
* **Standard**: `-std=c++98` 플래그를 추가해도 완벽하게 컴파일되어야 합니다.

### 2. Naming Conventions
* **Directories**: `ex00`, `ex01`, ..., `exn`
* **Class Names**: **UpperCamelCase** 형식을 따릅니다.
* **Files**: 헤더 파일과 소스 파일명은 반드시 클래스 이름과 일치해야 합니다. (예: `Fixed.hpp`, `Fixed.cpp`)

### 3. Forbidden & Allowed
* **C 함수 사용 금지**: `*printf()`, `*alloc()`, `free()` 사용 시 **0점** 처리됩니다.
* **STL 제한**: Module 08 이전까지 컨테이너(vector, list, map 등)와 알고리즘 헤더 사용이 금지됩니다.
* **🚨 [NEW] Orthodox Canonical Class Form (OCCF)**: Module 02부터 명시된 모든 클래스는 반드시 다음 4가지를 포함하는 **정통 표준 형태**로 작성되어야 합니다.
  1. 기본 생성자 (Default constructor)
  2. 복사 생성자 (Copy constructor)
  3. 복사 대입 연산자 (Copy assignment operator)
  4. 소멸자 (Destructor)

## 🏗️ Exercise 00: My First Class in Orthodox Canonical Form
정통 표준 클래스 형태(OCCF)를 준수하여 빈 껍데기의 고정 소수점 클래스를 처음으로 설계해보는 단계입니다.

### Objectives
* 정수를 저장할 `_val` 변수와, 소수부 비트 수를 나타내는 `const static int _bits = 8`을 가진 `Fixed` 클래스를 만듭니다.
* 기본 생성자, 복사 생성자, 복사 대입 연산자, 소멸자가 호출될 때마다 메시지를 출력하도록 하여 객체의 생명 주기와 복사 과정을 시각적으로 확인합니다.
* `getRawBits()`와 `setRawBits()` 멤버 함수를 구현합니다.

### 🚀 Usage & Testing (사용 및 테스트 방법)

1. **컴파일 (Compilation)**
	```bash
	$ cd ex00
	$ make
	```
2. **실행 (Execution)**
	```bash
	$ ./fixed
	```
	기본 생성자, 복사 생성자, 복사 대입 연산자가 호출되는 순서가 과제의 예시 출력과 정확히 일치하는지 확인합니다.

### 💡 Deep Dive

#### **Q1. 정통 표준 클래스 형태(Orthodox Canonical Form)란 무엇이며, 왜 써야 하나요?**
* **답변**: C++에서 객체를 안전하게 복사하고 대입하고 소멸시키기 위해 권장되는 클래스 설계 표준입니다. 컴파일러가 기본적으로 만들어주는 복사 생성자나 대입 연산자는 **얕은 복사(Shallow Copy)**를 수행하므로, 내부에 동적 할당된 포인터가 있을 경우 치명적인 메모리 오류(Double Free 등)가 발생할 수 있습니다. 이를 방지하고 객체의 생명 주기를 프로그래머가 완벽히 통제하기 위해 반드시 4가지 요소를 직접 명시하는 습관을 들여야 합니다.

#### **Q2. 복사 생성자(`Fixed(const Fixed& src)`)와 복사 대입 연산자(`operator=`)의 차이는?**
* **답변**:
    * **복사 생성자**: 새로 태어나는 객체가 기존 객체를 본떠서 만들어질 때 호출됩니다. (예: `Fixed b(a);` 또는 `Fixed b = a;`)
    * **복사 대입 연산자**: 이미 태어나서 존재하고 있는 객체에 기존 객체의 값을 덮어씌울 때 호출됩니다. (예: `c = b;`)

#### **Q3. 복사 대입 연산자에서 `if (this != &rhs)` 검사를 하는 이유는 무엇인가요?**
* **답변**: **자기 대입(Self-assignment)**을 방지하기 위해서입니다. `a = a;` 처럼 자기 자신을 대입할 경우, 만약 내부에서 메모리를 해제하고 다시 할당하는 로직이 있다면 원본 데이터가 날아가는 등 예상치 못한 오류가 발생할 수 있기 때문에 주소를 비교하여 같으면 아무 작업도 하지 않고 `*this`를 반환합니다.

## 🧮 Exercise 01: Towards a more useful fixed-point number class
부동 소수점(float)과 정수(int)를 고정 소수점(Fixed-point)으로 변환하는 생성자를 추가하고, 연산자 오버로딩을 통해 C++의 입출력 스트림과 호환되게 만드는 단계입니다.

### Objectives
* `int`와 `float`를 인자로 받아 고정 소수점 값으로 변환하는 생성자를 추가합니다.
* 고정 소수점 값을 다시 `int`와 `float`로 되돌리는 `toInt()`, `toFloat()` 함수를 구현합니다.
* `std::cout <<` 에 고정 소수점 객체를 바로 출력할 수 있도록 `operator<<`를 오버로딩합니다.

### 🚀 Usage & Testing (사용 및 테스트 방법)

1. **컴파일 (Compilation)**
	```bash
	$ cd ex01
	$ make
	```
2. **실행 (Execution)**
	```bash
	$ ./fixed
	```
	정수와 부동 소수점이 고정 소수점으로 잘 변환되어 저장되는지, 그리고 `toInt`, `toFloat`를 통해 원상 복구 시 오차가 얼마나 발생하는지 확인합니다.

### 💡 Deep Dive

#### **Q1. 왜 `float`나 `double`을 쓰지 않고 굳이 고정 소수점(Fixed-point)을 만드나요?**
* **답변**: 부동 소수점(`float`)은 아주 큰 수나 아주 작은 수를 표현하는 데는 유리하지만, 구조상 미세한 정밀도 오차(예: `0.1 + 0.2 != 0.3`)가 발생하고 연산 속도가 상대적으로 느립니다. 반면 고정 소수점은 소수부의 자리수를 고정해두고 내부적으로는 단순한 **정수 연산**을 수행하므로, 금융 시스템이나 임베디드 장비, 그래픽스처럼 **성능과 예측 가능한 정밀도**가 중요한 곳에서 유용하게 쓰입니다.

#### **Q2. 정수를 고정 소수점으로 변환할 때 비트 시프트 연산(`<< _bits`)을 하는 이유는?**
* **답변**: 소수부를 표현하기 위해 예약해 둔 8비트(`_bits`)만큼 정수를 왼쪽으로 밀어버리는 것입니다. 즉, 수학적으로는 `값 * 2^8 (256)`을 하는 것과 같으며, 이 밀려난 빈 8비트의 공간을 나중에 소수점 이하 값을 저장하는 용도로 사용합니다.

#### **Q3. `float`를 고정 소수점으로 변환할 때는 어떻게 하나요?**
* **답변**: `float` 값에 `1 << _bits` (즉, 256)를 곱한 뒤, 반올림 함수인 `roundf()`를 사용하여 가장 가까운 정수로 만들어 `_val`에 저장합니다. 반대로 `toFloat()`로 되돌릴 때는 저장된 `_val`을 `256.0f`로 나누어줍니다.

#### **Q4. `operator<<` 는 왜 클래스 내부가 아닌 외부에 선언하나요?**
* **답변**: `operator<<`의 왼쪽 피연산자는 `Fixed` 객체가 아니라 `std::ostream` 객체(예: `std::cout`)입니다. 클래스 내부에 멤버 함수로 연산자를 오버로딩하면 왼쪽 피연산자가 반드시 해당 클래스의 객체(`this`)가 되어버리므로 `a << std::cout;` 처럼 어색하게 써야 합니다. 따라서 `std::cout << a;` 형태를 유지하기 위해 전역 함수로 오버로딩해야 합니다.

## ⚙️ Exercise 02: Now we're talking
비교 연산자, 산술 연산자, 증감 연산자를 완벽하게 오버로딩하여, 커스텀 클래스인 `Fixed` 객체를 마치 C++의 기본 자료형(int, float)처럼 자연스럽게 다룰 수 있게 만드는 단계입니다.

### Objectives
* **비교 연산자**: `>`, `<`, `>=`, `<=`, `==`, `!=`
* **산술 연산자**: `+`, `-`, `*`, `/`
* **증감 연산자**: 전위(pre) `++`, `--` / 후위(post) `++`, `--`
* **Min/Max 함수**: 두 고정 소수점 중 더 작거나 큰 값을 반환하는 `min`, `max` 멤버 함수 구현 (const 오버로딩 포함)

### 🚀 Usage & Testing (사용 및 테스트 방법)

1. **컴파일 (Compilation)**
	```bash
	$ cd ex02
	$ make
	```
2. **실행 (Execution)**
	```bash
	$ ./fixed
	```
	증감 연산자의 결과, 곱셈/나눗셈 결과가 상식적인 수학 연산과 일치하는지 확인합니다.

### 💡 Deep Dive

#### **Q1. 산술 연산자 중 곱셈(`*`)과 나눗셈(`/`)은 어떻게 구현하나요?**
* **답변**: 단순히 `_val`끼리 곱하거나 나누면 단위가 꼬이게 됩니다. (예: `256 * 256 = 65536`). 따라서 가장 쉬운 방법은 두 객체를 `toFloat()`로 부동 소수점으로 바꾼 뒤 연산하고, 그 결과로 새로운 `Fixed` 객체를 생성하여 반환하는 것입니다.
*(물론 내부적으로 64비트 정수를 활용해 비트 시프트로 해결하는 방법이 성능상 더 좋지만, 과제 통과를 위해서는 float 캐스팅이 가장 안전합니다.)*

#### **Q2. 전위 증감(`++a`)과 후위 증감(`a++`)은 오버로딩 시 어떻게 구분하나요?**
* **답변**: 문법적으로 구별하기 위해 후위 증감 연산자는 매개변수로 **더미(dummy) `int`**를 받도록 약속되어 있습니다.
  * 전위: `Fixed& operator++(void)`
  * 후위: `Fixed operator++(int)`

#### **Q3. 왜 전위 증감은 참조(`&`)를 반환하고, 후위 증감은 값(객체 복사본)을 반환하나요?**
* **답변**:
  * **전위(`++a`)**: 객체 자신의 값을 즉시 증가시키고 바뀐 자기 자신을 그대로 반환하면 되므로 성능을 위해 참조(`Fixed&`)를 반환합니다.
  * **후위(`a++`)**: 값을 먼저 반환한 뒤에 증가해야 하므로, 증가되기 전의 상태를 저장할 임시 객체를 만들어야 합니다. 이 임시 객체는 함수가 끝나면 사라지는 지역 변수이므로, 참조가 아닌 **복사본(값)**을 반환해야 합니다. 따라서 후위 연산이 전위 연산보다 성능 비용이 미세하게 더 듭니다.

#### **Q4. 멤버 함수 뒤에 붙는 `const`와 `static` 함수는 각각 어떤 의미인가요?**
* **답변**:
    * **함수 뒤 `const`**: "이 함수는 객체의 상태(`_val`)를 절대 바꾸지 않는다"는 보증수표입니다. `const Fixed` 객체는 이 `const`가 붙은 함수들만 호출할 수 있습니다.
    * **`static` 함수**: 특정 객체가 아니라 클래스 자체에 소속된 도구입니다. `Fixed::max(a, b)`처럼 객체 없이 호출하며, 내부에서 `this` 포인터를 사용할 수 없습니다.

#### **Q5. 클래스 내부에서 `_val`에 접근할 때 직접 접근과 `getRawBits()` 호출 중 무엇이 더 좋나요? (캡슐화 쟁점)**
* **답변**:
    * **나(`this`)의 데이터**: `this->_val`로 직접 접근하는 것이 함수 호출 오버헤드를 줄여 효율적이며, 같은 클래스 내부이므로 캡슐화를 해치지 않습니다.
    * **남(`rhs`)의 데이터**: 비록 같은 타입이라도 `rhs.getRawBits()`를 통해 공식 인터페이스를 거치는 것이 객체 간의 예의(캡슐화)를 지키는 더 좋은 설계 습관입니다.

#### **Q6. 연산자 오버로딩에서 `this`와 `rhs`는 어떻게 구분하나요?**
* **답변**: 이항 연산자(예: `a > b`)를 멤버 함수로 구현할 때:
    * **`this`**: 연산자 **왼쪽**의 피연산자(`a`)를 가리킵니다.
    * **`rhs`**: 연산자 **오른쪽**의 피연산자(`b`, Right Hand Side)를 가리킵니다.

#### **Q7. 0으로 나누는 연산에 대해 예외 처리가 꼭 필요한가요?**
* **답변**: 서브젝트(PDF)에서 "프로그램이 중단되어도 무방하다"고 명시했으므로 필수사항은 아닙니다. Sisung님의 구현 방식(`toFloat`)으로는 시스템이 터지는 대신 부동 소수점 표준에 따라 `-8.38861e+06` 같은 오버플로우 된 값이 출력됩니다.

## 📐 Exercise 03: BSP (Binary Space Partitioning)
지금까지 만든 강력한 `Fixed` 클래스를 활용하여, 2D 공간의 점(`Point`) 객체를 만들고 특정 점이 삼각형 내부에 있는지 판별하는 수학적 알고리즘을 구현하는 단계입니다.

### Objectives
* 고정 소수점을 사용하는 2D 좌표 클래스 `Point`를 작성합니다. (x, y 값은 상수로 보호됨)
* BSP(이진 공간 분할) 원리 또는 외적(Cross Product) / 넓이(Barycentric coordinates) 방식을 사용하여 `bsp` 함수를 완성합니다.
* `bool bsp( Point const a, Point const b, Point const c, Point const point);`

### 🚀 Usage & Testing (사용 및 테스트 방법)

1. **컴파일 (Compilation)**
	```bash
	$ cd ex03
	$ make
	```
2. **실행 (Execution)**
	```bash
	$ ./bsp
	```
	내부에 있는 점(True), 외부에 있는 점(False), 선상이나 꼭짓점에 있는 점(False 처리) 등 다양한 엣지 케이스를 직접 `main.cpp`에 작성하여 테스트해야 합니다.

### 💡 Deep Dive

#### **Q1. 점이 삼각형 내부에 있는지 어떻게 판별하나요? (추천: 외적 방식)**
* **답변**: 삼각형의 세 변을 따라가면서(AB, BC, CA) 각 선분과 테스트할 점(P) 사이의 방향성(Cross Product)을 확인하는 방법이 가장 널리 쓰입니다.
벡터 AB와 AP, BC와 BP, CA와 CP의 외적을 각각 구했을 때, 그 결과값의 부호(+, -)가 **모두 같으면** 점은 삼각형 내부에 존재합니다. 단 하나라도 부호가 다르면 외부에 있는 것입니다. 결과값이 0이라면 점이 선상(Edge)이나 꼭짓점(Vertex)에 있는 것이며 과제 요구사항에 따라 내부에 포함되지 않는 것(False)으로 처리해야 합니다.

#### **Q2. `Point` 클래스에서 `x`, `y`를 `Fixed const`로 만든 이유는 무엇인가요?**
* **답변**: 한 번 생성된 점의 좌표는 불변(Immutable)해야 한다는 설계 의도입니다. `const` 멤버 변수는 생성자 본문에서 대입할 수 없으므로, 반드시 **초기화 리스트(Initialization List)**를 사용하여 객체가 태어남과 동시에 값을 할당해야 합니다.
