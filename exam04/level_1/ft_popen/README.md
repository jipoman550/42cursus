# ft_popen 함수 상세 가이드

## 1. 목적 (Purpose)

`ft_popen` 함수는 C 프로그램에서 외부 명령어를 실행하고, 해당 명령어의 표준 입력(stdin) 또는 표준 출력(stdout)과 통신할 수 있는 간결한 인터페이스를 제공합니다. 이 함수는 프로세스 생성(`fork`), 프로세스 간 통신(`pipe`), I/O 재지정(`dup2`, `execvp`)과 같은 복잡한 과정을 추상화하여, 외부 커맨드라인 유틸리티를 C 애플리케이션에 쉽게 통합할 수 있도록 돕습니다.

## 2. 주요 기능 (Functionality)

`ft_popen`은 특정 실행 파일(`file`)을 주어진 인자(`argv`)와 함께 실행하며, 지정된 `type`에 따라 입력 또는 출력에 사용할 수 있는 **파일 디스크립터(File Descriptor)**를 반환합니다.

* **`file`**: 실행할 프로그램의 이름 (예: `"ls"`, `"grep"`).
* **`argv`**: 명령어와 인자들을 담은 NULL 종료 문자열 배열. `execvp`에 직접 전달됩니다. (예: `(char *const []){"ls", "-l", NULL}`)
* **`type`**:
* `'r'`: 명령어를 실행하고, 해당 명령어의 **표준 출력(stdout)**과 연결된 파일 디스크립터를 반환합니다. 이를 통해 프로그램은 실행된 명령어의 결과 데이터를 읽어올 수 있습니다.
* `'w'`: 명령어를 실행하고, 해당 명령어의 **표준 입력(stdin)**과 연결된 파일 디스크립터를 반환합니다. 이를 통해 프로그램은 실행된 명령어에게 데이터를 보낼 수 있습니다.

> [!CAUTION]
> 실행 중 오류(파이프 생성 실패, 포크 실패, 명령어 실행 실패 등)가 발생하거나 잘못된 파라미터가 입력되면 함수는 `-1`을 반환합니다.

## 3. 동작 원리 (How it Works)

`ft_popen`은 기능을 구현하기 위해 내부적으로 다음과 같은 유닉스 시스템 콜을 조합합니다.

### 1단계: `pipe(fds)`

프로세스 간 단방향 통신 채널을 생성합니다. `fds[0]`은 읽기용, `fds[1]`은 쓰기용 디스크립터가 됩니다.

### 2단계: `fork()`

부모 프로세스를 복제하여 새로운 자식 프로세스를 생성합니다. 자식 프로세스에서는 `0`을, 부모 프로세스에서는 자식의 PID를 반환합니다.

### 3단계: 자식 프로세스 내부 (`pid == 0`)

* **`dup2()`를 이용한 I/O 재지정**:
* `type == 'r'`이면: 자식의 표준 출력을 파이프의 쓰기 끝(`fds[1]`)으로 연결합니다.
* `type == 'w'`이면: 자식의 표준 입력을 파이프의 읽기 끝(`fds[0]`)에서 받도록 연결합니다.

* **미사용 디스크립터 닫기**: 연결이 완료되면 사용하지 않는 파이프의 양 끝단을 닫습니다.
* **`execvp(file, argv)`**: 자식 프로세스를 지정된 외부 프로그램 이미지로 교체하여 실행합니다.

### 4단계: 부모 프로세스 내부 (`pid > 0`)

* **미사용 디스크립터 닫기**:
* `type == 'r'`이면: 쓰기 끝(`fds[1]`)을 닫고 **읽기 끝(`fds[0]`)**을 반환합니다.
* `type == 'w'`이면: 읽기 끝(`fds[0]`)을 닫고 **쓰기 끝(`fds[1]`)**을 반환합니다.

* 반환된 디스크립터는 부모 프로세스에서 `read()`나 `write()`를 통해 사용됩니다.

---

## 4. 왜 `ft_popen`을 사용하는가? (Benefits)

* **IPC의 단순화**: `pipe`, `fork`, `dup2` 등의 복잡한 시스템 콜을 직접 관리할 필요가 없습니다. 외부 명령어를 마치 일반 파일을 다루듯이 조작할 수 있습니다.
* **유연한 통합**: `ls`나 `grep` 같은 강력한 도구의 결과물을 직접 캡처하거나 데이터를 전달하기에 매우 용이합니다.
* **가독성**: 수동으로 IPC를 구현하는 것보다 코드가 훨씬 깔끔하고 유지보수가 쉬워집니다.

## 5. 예제 코드(`main.c`) 설명

### 시나리오 1: 명령어 결과 읽기 (`'r'` 모드)

`ls -l` 명령어를 실행하고 그 출력 내용을 버퍼에 담아 출력합니다.

```c
int fd = ft_popen("ls", args_ls, 'r');
while ((n = read(fd, buf, 1023)) > 0) {
    // 읽어온 내용 처리
}
close(fd);
```

### 시나리오 2: 명령어에 입력 보내기 (`'w'` 모드)

`grep hello` 명령어를 실행하고, 부모 프로세스에서 특정 문자열을 입력으로 던져줍니다.

```c
int fd = ft_popen("grep", args_grep, 'w');
write(fd, "hello world\n", 12);
close(fd); // 중요: close를 해야 grep이 입력이 끝났음을 인지하고 종료됩니다.

```

---

> [!IMPORTANT]
> **자원 관리 주의사항**: `ft_popen`이 반환한 파일 디스크립터는 반드시 호출한 쪽(Caller)에서 `close()` 해줘야 합니다. 이는 자원 누수를 방지하고, 자식 프로세스에 입력 종료 신호를 보내기 위해 필수적입니다.
