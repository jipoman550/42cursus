# 🕹️ cub3D: My first RayCaster with miniLibX

## 📌 프로젝트 목적

* **Wolfenstein 3D의 재림**: 세계 최초의 FPS 게임인 Wolfenstein 3D에서 영감을 받아, 레이캐스팅(Ray-casting) 원리를 이용한 3D 미로 탐색 게임을 구현합니다.
* **수학적 도구의 활용**: 삼각함수와 기하학적 알고리즘(DDA)을 사용하여 2D 지도를 1인칭 시점의 3D 그래픽으로 변환하는 효율적인 알고리즘을 설계합니다.
* **프로그래밍 역량 강화**: C 언어의 엄격한 활용, 알고리즘 최적화, 정보 검색 능력 및 그래픽 라이브러리(miniLibX) 컨트롤 능력을 배양합니다.

## 🧠 과제 개요

### Mandatory Part (필수 구현)

* **레이캐스팅 엔진**: 사용자의 시점에 따라 실시간으로 변하는 3D 뷰를 생성해야 합니다.
* **벽 텍스처**: 동서남북 방향에 따라 서로 다른 텍스처가 표시되어야 합니다.
* **바닥 및 천장**: 설정 파일에서 지정한 RGB 색상으로 채워져야 합니다.
* **사용자 제어**:
	* `W`, `A`, `S`, `D`: 미로 내 이동.
	* `좌/우 화살표`: 시점 회전.
	* `ESC` 및 `Red Cross`: 프로그램의 깔끔한 종료.
* **맵 파싱**: `.cub` 확장자 파일을 읽어 맵의 유효성(벽으로 둘러싸여 있는지 등)을 검사합니다.

### Bonus Part (추가 구현)

* **벽 충돌**: 벽을 통과하지 못하도록 물리 엔진을 구현합니다.
* **미니맵**: 현재 위치와 미로의 구조를 보여주는 2D 맵을 표시합니다.
* **문(Doors)**: 열고 닫을 수 있는 문 기능을 추가합니다.
* **애니메이션 스프라이트**: 움직이는 캐릭터나 오브젝트를 추가합니다.
* **마우스 회전**: 마우스를 사용하여 시점을 회전시킵니다.

## 🏗️ 아키텍처

레이캐스팅의 핵심은 플레이어의 위치로부터 화면의 가로 픽셀만큼 레이(Ray)를 쏘아 벽까지의 거리를 계산하는 것입니다.

* **DDA (Digital Differential Analysis)** 알고리즘: 레이가 그리드 선을 지날 때마다 확인하여 가장 가까운 벽을 찾습니다.
* **수직 거리 계산**: 어안 렌즈 효과(Fish-eye effect)를 방지하기 위해 플레이어의 평면에 수직인 거리를 사용합니다.

## 🛠️ 디버깅 및 문제 해결 히스토리

* **Memory Leaks**: `malloc`으로 할당된 모든 힙 메모리는 종료 시 적절히 `free`되어야 합니다. `valgrind`를 사용하여 누수를 추적합니다.
* **Segmentation Fault**: 맵 파싱 중 잘못된 접근이나 `NULL` 포인터 참조를 주의해야 합니다.
* **Norm Errors**: 42 Norm v4.1 규칙(함수 당 25라인, 변수 5개 등)을 철저히 준수합니다.

## 📂 디렉토리 구조

```text
.
├── includes/       # 헤더 파일 (*.h)
├── libft/          # 사용자 정의 라이브러리
├── maps/           # 테스트용 .cub 맵 파일
├── srcs/           # 소스 파일 (*.c)
│   ├── mandatory/  # 필수 파트 코드
│   └── bonus/      # 보너스 파트 코드
├── textures/       # 벽 텍스처 이미지 (XPM 형식)
└── Makefile        # 빌드 스크립트

```

## 빌드 및 실행

```bash

make all

# Bonus
make bonus

```

### 실행

```bash
./cub3D maps/test.cub

```
